# MonthLow 选股策略 · 四阶段逻辑说明

> 文件：`MonthLow.py`  
> 策略目标：筛选出**昨日（T-1日）创近N日新低、今日（T日）最低价高于昨日最低价**的主板股票，捕捉短期低点反弹机会。

---

## 整体流程

```
第一阶段  →  第二阶段  →  [缓存预热]  →  第三阶段  →  第四阶段  →  落库 & 输出
生成股票池     质量过滤                    低点筛选       反弹确认
```

---

## 第一阶段：生成主板股票池

**函数**：`get_stock_list()`

### 目标
从沪深两市生成主板股票代码列表，排除创业板、科创板、北交所。

### 股票池范围

| 市场 | 代码范围 | 说明 |
|------|----------|------|
| 深证主板 | `sz000001` ~ `sz002999` | 含原中小板（002xxx），排除创业板（300xxx+） |
| 上证主板 | `sh600000` ~ `sh603999` | 排除科创板（688xxx+） |

### 输出
约 **6973 只**股票代码（深证 2999 + 上证 4000）。

---

## 第二阶段：质量过滤

**函数**：`filter_by_name(codes)`

### 目标
通过腾讯行情接口批量获取股票名称、价格、品种类型，过滤掉不适合交易的品种。

### 数据来源
`http://qt.gtimg.cn/q={codes}` — 腾讯财经实时行情接口，每次最多 100 只，多线程并发（20 workers）。

### 过滤规则

| 过滤条件 | 判断依据 | 说明 |
|----------|----------|------|
| ST / \*ST 股 | 名称含 `ST`（不区分大小写） | 风险股，排除 |
| 退市股 | 名称含「退」或以 `PT` 开头 | 已退市或即将退市 |
| 非 A 股 | 品种类型字段（field[61]）≠ `GP-A` | 排除可转债、债券、指数等 |
| 低价股 | 价格 ≤ 1 元 | 问题股或无成交 |

### 输出
过滤后约 **2000~2500 只**有效 A 股。

---

## 缓存预热（第三阶段前置）

**函数**：`_warm_up_cache(codes, lookback, workers)`

### 目标
在第三阶段开始前，批量将所有股票的日线数据拉取并写入本地 MySQL 缓存（`t_stock_daily` 表），使第三阶段直接读库而不发 HTTP 请求，大幅提升速度。

### 执行策略（读写分离，消除死锁）

```
第①步：并发 HTTP 拉取（100 workers）
  ├─ 检查每只股票的缓存状态
  │    ├─ 缓存条数足够 且 日期是最新交易日 → 跳过（命中缓存）
  │    ├─ 交易时间内（09:00~15:00）且缓存是今天 → 强制回源（当天数据实时变动）
  │    └─ 其他情况 → 加入待拉取列表
  └─ 对待拉取列表并发调用 Ashare API，结果存内存

第②步：单线程批量写入数据库
  └─ 顺序写入，彻底消除高并发死锁（MySQL 1213 错误）
```

### 缓存有效性判断

| 条件 | 结果 |
|------|------|
| 缓存条数 < N+10 | 需要回源 |
| 缓存最新日期 < 最近交易日 | 需要回源 |
| 缓存最新日期 = 今天 且 当前在交易时间 | 需要回源（盘中数据实时变动） |
| 其他 | 命中缓存，跳过 |

> **最近交易日**通过拉取上证指数（sh000001）最新一条日线确定，进程内缓存，只查一次。

## 第三阶段：T-1 日创近 N 日新低筛选

**函数**：`check_month_low(code)`，由 `pick_month_low_stocks()` 并发调用（100 workers）

### 目标
找出**昨日（T-1日）最低价等于近 N 个交易日最低价**的股票，即昨日创了近期新低。

### 数据来源
`_get_daily(code, frequency='1d', count=N+10)` — 优先读本地缓存（预热后全部命中），缓存缺失时回源 Ashare API。

### 筛选逻辑

```
取近 N+10 条日线数据
  ├─ T 日  = df.iloc[-1]   （今日，最新一条）
  ├─ T-1 日 = df.iloc[-2]  （昨日）
  └─ 近 N 日 = df.tail(N)  （含 T-1 日，不含 T 日）

筛选条件：
  abs(T-1日最低价 - 近N日最低价) <= 0.01
  即：T-1 日最低价 ≈ 近 N 日最低价（T-1 日创了近 N 日新低）
```

> 允许 0.01 元误差，处理浮点四舍五入问题。

### 可配置参数

```python
CONFIG["filter"]["lookback_days"] = 10  # 默认 10 日，可通过 --lookback N 覆盖
```

### 输出
满足条件的股票列表，包含字段：`code`、`current_price`、`t_low`、`t_1_low`、`period_low`、`lookback_days`。

## 第四阶段：T 日最低价高于 T-1 日最低价（反弹确认）

**函数**：`filter_t_low_above_t_1_low(results)`

### 目标
在第三阶段结果中，进一步筛选**今日最低价高于昨日最低价**的股票，确认今日没有继续创新低，出现反弹迹象（含十字星）。

### 筛选逻辑

```
筛选条件：T 日最低价 > T-1 日最低价
```

### 附加操作
批量调用腾讯接口补充股票名称（第三阶段为节省时间未获取名称）。

### 输出字段

| 字段 | 说明 |
|------|------|
| `code` | 股票代码（含 sz/sh 前缀） |
| `name` | 股票名称 |
| `current_price` | T 日收盘价 |
| `t_low` | T 日最低价 |
| `t_1_low` | T-1 日最低价 |
| `price_vs_t1_low_pct` | (当前价 - T-1最低) / T-1最低 × 100% |
| `t_low_vs_t1_low_pct` | (T最低 - T-1最低) / T-1最低 × 100% |
| `lookback_days` | 本次使用的回看天数 |

> 结果按 `price_vs_t1_low_pct` 从低到高排序（偏离昨日低点越小排越前）。

---

## 落库

**函数**：`upsert_month_low_results(scan_date, rows)`（`db.py`）

- 写入表：`t_month_low_result`
- 主键冲突策略：`ON DUPLICATE KEY UPDATE`，同一天同一股票覆盖更新
- 可通过 Web 页面（`/month_low`）按日期查看历史结果

---

## 运行方式

```bash
# 全市场扫描（默认 lookback=40）
python MonthLow.py

# 指定回看天数
python MonthLow.py --lookback 10

# 测试指定股票
python MonthLow.py --code sz000001 --code sh600036

# 从文件读取股票列表
python MonthLow.py --file my_stocks.txt
```

## 性能说明

| 阶段 | 耗时估算 | 主要瓶颈 |
|------|----------|----------|
| 第一阶段 | < 1 秒 | 纯内存循环 |
| 第二阶段 | 10~20 秒 | 腾讯接口 HTTP 请求（20 并发） |
| 缓存预热（首次） | 60~120 秒 | Ashare API HTTP 请求（100 并发） |
| 缓存预热（热缓存） | < 5 秒 | 全部命中缓存，跳过 HTTP |
| 第三阶段 | 5~15 秒 | 读 MySQL 缓存（100 并发） |
| 第四阶段 | < 1 秒 | 内存过滤 + 一次腾讯接口 |
