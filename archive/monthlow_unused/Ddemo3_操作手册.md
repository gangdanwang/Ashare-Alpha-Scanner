# `Ddemo3.py` 操作手册

本脚本用于**A 股盘中快速选股**，包含两阶段筛选：

- **第一阶段（实时粗筛）**：基于腾讯 `qt.gtimg.cn` 的实时字段（涨跌幅/量比/换手率/流通市值）做快速过滤。
- **第二阶段（14:00 策略）**：拉取 5 分钟 K 线（`get_price(..., frequency='5m')`），判断 14:00 之后是否满足策略阈值。

脚本同时提供**全市场模式**和**快速测试模式（单股/多股）**。

---

## 运行前准备

- **Python 环境**：Python 3.10+（脚本用到了类型标注 `str | None`）。
- **依赖**：`requests`、`pandas`，以及项目内的 `Ashare.get_price` 可正常工作。
- **网络**：需要能访问数据源（腾讯实时接口、K 线数据源）。

---

## 运行方式

### 1) 全市场选股（默认）

不带任何参数时，脚本会生成股票池并全市场筛选：

```bash
python Ddemo3.py
```

输出：
- 终端打印：批次数、第一阶段数据量、粗筛剩余数、最终结果表、耗时
- 保存 CSV：`选股结果_高性能.csv`

---

### 2) 快速测试：单只股票

```bash
python Ddemo3.py --code sz002368
```

说明：
- 会只对该股票跑两阶段，并在终端打印两条关键日志：
  - `是否通过第一阶段： ...`
  - `是否通过第二阶段： ...`

---

### 3) 快速测试：多只股票（两种写法）

#### 写法 A：逗号分隔（`--codes`）

```bash
python Ddemo3.py --codes "sz002368,sh600681,sz000001"
```

#### 写法 B：重复传参（`--code` 可传多次）

```bash
python Ddemo3.py --code sz002368 --code sh600681 --code sz000001
```

输出：
- 对每个代码都会打印两条阶段日志（第一阶段/第二阶段是否通过）
- 汇总输出为一个表，并写入 `选股结果_高性能.csv`
- 汇总表里额外包含：
  - `passed_stage1`
  - `passed_stage2`

---

## 参数说明

- `--code <CODE>`
  - 可重复传入多次
  - 示例：`--code sz002368 --code sh600681`
- `--codes "<CODE1,CODE2,...>"`
  - 逗号分隔的批量代码
  - 示例：`--codes "sz002368,sh600681"`

> 同时传了 `--code` 和 `--codes` 时，会合并为同一组代码一起测试。

---

## 筛选规则说明（重要）

### 第一阶段 `fast_filter(info)`

当前条件为：
- 涨跌幅：`3 < 涨跌幅 < 7`
- 量比：`量比 > 1.8`
- 换手率：`3 < 换手率 < 12`
- 流通市值：`50 < 流通市值 < 300`

**注意：流通市值单位**

腾讯接口字段 `流通市值` 在本脚本按 **“亿元”** 处理，因此阈值写成 `50 ~ 300`（即 50–300 亿）。

---

### 第二阶段 `after_14_filter(code)`

- 拉取 5 分钟 K 线：`count=50`
- 取时间 >= 14:00 的数据段
- 判断从 14:00 第一根到当前最后一根的涨幅是否大于 `1.2%`

---

## 输出文件

- **CSV**：`选股结果_高性能.csv`
  - 全市场模式：只包含最终通过两阶段的股票
  - 快速测试模式：包含你测试的所有股票，并附带 `passed_stage1/passed_stage2`

## 常见问题排查

### 1) `unrecognized arguments: --codes`

原因：脚本未支持该参数（旧版本）。

解决：更新到当前版本后再运行：

```bash
python Ddemo3.py --codes "sz002368,sh600681"
```

---

### 2) 提示 `实时数据获取失败：<code>`

可能原因：
- 网络访问受限/超时
- 数据源临时不可用
- 代码格式不正确（建议使用 `sz`/`sh` 前缀）

建议：
- 换个时间重试
- 确认代码格式：例如 `sz002368`、`sh600681`

---

### 3) 第一阶段始终 False

常见原因：
- 阈值过严（例如量比、换手率、涨跌幅不在区间）
- **单位误用**：流通市值应按“亿元”比较（本脚本已按亿元阈值修正）

---

## 回测（2025 年）说明

本仓库额外提供一个**不改动 `Ddemo3.py`** 的回测脚本：

- `backtest_ddemo3_2025.py`
- 配置：`backtest_config.py`

### 重要限制（当前数据源）

由于 `Ashare.get_price()` 的分钟线接口无法按日期拉取 2025 年任意一天的 14:30/9:30 分钟K，本回测采用**日线近似**：

- 14:30 买入 ≈ 当日收盘价 `close`
- 次日 9:30 卖出 ≈ 次日开盘价 `open`

### 运行方式

先在 `backtest_config.py` 里配置：
- `universe`：回测股票列表（建议先填少量，避免太慢）
- `daily_fast_filter`：近似筛选阈值

运行：

```bash
python backtest_ddemo3_2025.py
```

输出：
- `回测交易明细_2025.csv`
- `回测每日汇总_2025.csv`
